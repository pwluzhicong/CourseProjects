Data in the XML file

The file format has to store information about meals.

You are provided with a sample file containing the data. Please note that you have to develop a schema file that matches the description below, schemas that accept the sample file but do not conform to the requirements below will not be accepted.

The document’s root element is called garden. Inside the element there are three kinds of elements: color, plant and insect. color and plant can be in any order. insect elements must be at the end (it means that after the first occurrence of insect only insect elements can be given).

The structure of the elements is as follows: - color - attribute name, lowercase string, required, must be unique across all colors - element red, a decimal number with exactly one digit in decimal part, the value must be in the range [0,1]. - element green, a decimal number with exactly one digit in decimal part, the value must be in the range [0,1]. - element blue, a decimal number with exactly one digit in decimal part, the value must be in the range [0,1]. - The child elements red, green, blue must be in exactly that order. - insect - attribute latinName – a string that starts with a capital letter and contains at least one space. Must be unique across both insects and plants. - attribute color, lowercase string, must be a name of a color defined in the document - attribute id – a string, must be unique across all plants - one or more month elements. Each month element contains one of the following values (10 possibilities): - January - March - April - May - June - July - August - September - October - November - plant - attribute name – a string - attribute latinName – a string that starts with a capital letter and contains at least one space. Must be unique across both insects and plants. - attribute color, lowercase string, must be a name of a color defined in the document - one or more month elements. Each month element contains one of the following values (10 possibilities): - January - March - April - May - June - July - August - September - October - November

Grading

  • (2 pts) Define XSD that will enforce the structure of the document. No keys and reference enforcement at this point. Modify XML so that elements are connected with XSD definition.
  • (1 pt) Do not define common part of plant and insect twice.
  • (1 pt) Write a program in C#, that will read the XML given as a command line argument and print to stdout all insects from file in the following format: Insect: {latinName} of color RGB: {R}/{G}/{B} where latingName is a value of the insects respective attribute and R,G,B are values of red, green, blue fields of the insect’s color
  • (1 pt) The solution is divided into separate project with class library and separate with console app.
  • (1 pt) Classes are automatically generated from XSD during compilation process.
  • (1 pt) If the provided XML does not conform to XSD, the program prints an error message and exits.
  • (2 pts) Enforce proper references from plants and insects to colors.
  • (1 pt) Enforce unique latinName.
